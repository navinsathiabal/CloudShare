# AWS S3 Setup Instructions

Follow these steps to configure your AWS environment for the Public File Sharing System.

## 1. Create an S3 Bucket
1. Open the Amazon S3 console.
2. Click **Create bucket**.
3. Enter a unique **Bucket name**.
4. Choose your preferred **AWS Region**.
5. Under **Object Ownership**, leave "ACLs disabled".
6. Under **Block Public Access settings for this bucket**, uncheck **Block all public access** (acknowledge the warning that the bucket will become public).
7. Click **Create bucket**.

## 2. Configure Bucket Policy (For Public Read)
To ensure that the links generated by the backend are accessible by anyone in a browser (public URLs), you need to add this explicit Bucket Policy.

1. Go to your new bucket.
2. Click the **Permissions** tab.
3. Scroll down to **Bucket policy** and click **Edit**.
4. Paste the following JSON (replace `YOUR-BUCKET-NAME` with your actual bucket name):

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
        }
    ]
}
```
5. Click **Save changes**.

## 3. Configure CORS (Cross-Origin Resource Sharing)
If you ever want your frontend to upload directly to S3 (or fetch files directly), you need a CORS configuration. For our backend-upload method, this ensures any direct interaction with S3 works smoothly for pre-signed URLs or image rendering.

1. In the **Permissions** tab, scroll down to **Cross-origin resource sharing (CORS)**.
2. Click **Edit**.
3. Paste the following configuration:

```json
[
    {
        "AllowedHeaders": [
            "*"
        ],
        "AllowedMethods": [
            "GET",
            "PUT",
            "POST",
            "HEAD"
        ],
        "AllowedOrigins": [
            "*"
        ],
        "ExposeHeaders": []
    }
]
```
*(Note: For production, change `AllowedOrigins` from `*` to your actual frontend domain).*
4. Click **Save changes**.

## 4. Setting up IAM Credentials
The Node.js backend requires permissions to `PUT` objects into S3.

1. Go to the **IAM** console in AWS.
2. Create an **IAM user** (e.g., `s3-upload-user`).
3. Attach the policy `AmazonS3FullAccess` (or create a custom inline policy specifically restricting `s3:PutObject` to your bucket).
4. Generate **Access Keys** for this user.
5. Copy the **Access Key ID** and **Secret Access Key**.

## 5. Running the Project Locally

1. **Backend:**
   - Rename `backend/.env.example` to `backend/.env`.
   - Fill in your `PORT`, `AWS_REGION`, `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, and `AWS_S3_BUCKET_NAME`.
   - Run `npm install` then `npm run start` (or `npx nodemon server.js`).

2. **Frontend:**
   - In `frontend`, run `npm install`.
   - Run `npm run dev`.
   - The React app will be available, usually at `http://localhost:5173`. Make sure the frontend `API_ENDPOINT` in `App.jsx` matches your backend port (default 5000).
